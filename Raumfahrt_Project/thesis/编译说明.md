## 编译步骤

### 方法1：命令行编译（推荐）

```bash
# 进入工程目录
cd d:/CodeSpace/CodeBuddy/model

# 第一次编译
xelatex main.tex

# 生成参考文献（需要安装 gbt7714 宏包）
bibtex main

# 再编译两次以更新交叉引用
xelatex main.tex
xelatex main.tex
```

### 方法2：使用 LaTeX 编辑器

1. 使用 TeXstudio、TeXworks 或 VS Code + LaTeX Workshop
2. 设置编译器为 **XeLaTeX**
3. 按 F5 或点击编译按钮

## 必需安装的宏包

确保您的 TeX 发行版（TeX Live 或 MiKTeX）已安装以下宏包：

- `ctex` - 中文支持
- `geometry` - 页面设置
- `titlesec` - 标题格式
- `titletoc` - 目录格式
- `fancyhdr` - 页眉页脚
- `caption` - 图表标题
- `booktabs` - 三线表
- `longtable` - 长表格
- `algorithm2e` - 算法排版
- `tikz` / `pgfplots` - 绘图
- `gbt7714` - 国标参考文献格式（可选，如未安装可改用 plain 样式）

### 安装 gbt7714 宏包

```bash
# TeX Live
tlmgr install gbt7714

# MiKTeX
miktex packages install gbt7714
```

如果无法安装 gbt7714，请修改 `chapters/references.tex`：
```latex
\bibliographystyle{plain}  % 改用通用样式
```

## 常见问题

### Q1: 编译报错 "gbt7714-numerical not found"
**解决**：安装 gbt7714 宏包，或修改 `chapters/references.tex` 使用 `\bibliographystyle{plain}`

### Q2: 中文显示为方框
**解决**：确保使用 XeLaTeX 编译，而非 pdfLaTeX

### Q3: 图片无法显示
**解决**：确保图片放在正确的 `figures/chX/` 目录下，并在正文中正确引用

### Q4: 目录不更新
**解决**：需要编译两次，或删除 `.aux` 文件后重新编译

## 输出文件

编译成功后将生成：
- `main.pdf` - 最终论文 PDF
- `main.aux` - 辅助文件（可删除）
- `main.log` - 日志文件（可删除）
- `main.toc` - 目录文件（可删除）
- `main.bbl` - 参考文献格式化文件（可删除）
